# Playbook System

Use this file when the team needs a repeatable situation-based operating procedure instead of generic advice.

## Playbook law

- A playbook exists for recurring situations with meaningful failure cost.
- A playbook is a decision path, not a theory note.
- If a workflow happens often and still depends on memory, it belongs in a playbook.

## Core playbooks

- Start a new game
- Define MVP
- Build first production architecture
- Design level config tables
- Prepare vertical slice
- Prepare release candidate
- Run soft launch review
- Triage live incident
- Review risky refactor

## Playbook format

- Trigger
- Preconditions
- Required inputs
- Required owners
- Steps
- Stop conditions
- Outputs
