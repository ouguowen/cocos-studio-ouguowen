# Project Memory System

Use this file when the team needs stable project facts, locked assumptions, current stage truth, accepted risks, or active decisions to persist across many conversations or work sessions.

## Memory law

- Memory stores project truth, not random notes.
- A remembered lie is worse than an unknown fact.
- Every memory item must be classified as confirmed, assumed, or open.

## Minimum memory buckets

- current stage
- current production mode
- current milestone
- classified game type
- MVP truth
- platform targets
- locked architecture decisions
- accepted risks
- blocked items
- next review date

## Memory rules

- Keep facts short and explicit.
- Separate confirmed facts from temporary assumptions.
- Expire stale assumptions when the project changes.
- Update memory after milestone, risk, scope, or platform decisions.

## First memory creation order

1. Record current stage and production mode.
2. Record classified game type.
3. Record current milestone purpose.
4. Record MVP truth and explicit non-goals.
5. Record platform targets and locked architecture decisions.
6. Record accepted risks and blocked items.
7. Record the next review date.
