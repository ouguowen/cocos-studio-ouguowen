# AI Game Studio v2.4 MCP Integration Validation

This document defines validation architecture for integrating AI Game Studio
with an external Cocos MCP Server. It is documentation only. It does not
implement MCP, duplicate MCP development, execute tools, automate Cocos, or
modify real Cocos project files.

## External Cocos MCP Server Integration

AI Game Studio should treat the Cocos MCP Server as an external execution layer.

AI Game Studio responsibilities:

- interpret human requests
- route planning through Agent governance
- select capability intent
- apply permission checks
- prepare execution proposals
- normalize tool results
- review validation output

External Cocos MCP Server responsibilities:

- expose Cocos Creator tool capabilities
- execute approved Cocos operations
- return structured results, warnings, errors, and generated file information

AI Game Studio must not duplicate the external MCP server. It should integrate
through an adapter boundary.

## Cocos Creator 3.8.8 Compatibility

Integration validation must confirm compatibility with Cocos Creator 3.8.8:

- scene hierarchy operations are editor-safe
- node transforms and parent-child operations are supported
- component add/remove/configure operations are supported
- prefab creation and instantiation respect Cocos references
- asset metadata and UUID references are preserved
- animation clips and state data are handled through safe editor operations
- UI Canvas, Widget, Layout, Button, Label, and panel operations are supported
- validation tools can inspect project, scene, asset, and runtime proof state

Compatibility proof must be explicit. A connected MCP server is not enough by
itself.

## MCP Tool Capability Mapping

Required capability groups:

- scene
- node
- component
- prefab
- animation
- builder
- validate

Mapping examples:

| Capability | Adapter Intent | External Cocos MCP Tool Family |
| --- | --- | --- |
| scene | create, open, save, inspect | scene tools |
| node | create, delete, rename, transform | node tools |
| component | add, remove, configure | component tools |
| prefab | create, instantiate, inspect | prefab tools |
| animation | create clip, bind state, inspect | animation tools |
| builder | build, package, output check | build tools |
| validate | hierarchy, console, preview, diff | validation tools |

The adapter should map AI Game Studio capability names to external tool
capabilities without hard-coding one provider into core policy.

## Integration Flow

```text
Human request
      |
      v
Planner Agent
      |
      v
Capability Registry
      |
      v
MCP Adapter
      |
      v
External Cocos MCP Server
      |
      v
Cocos Creator
```

Every step must preserve approved scope, permission boundary, proof target, and
rollback expectations.

## Validation Checks

Integration validation should check:

- server availability
- tool discovery output
- capability mapping completeness
- version compatibility
- permission filtering
- safe request translation
- normalized response format
- generated or modified file reporting
- error handling
- audit record completeness

Validation result should be PASS, NEEDS_REPAIR, BLOCKED, or FAIL.

## Safety

This document creates:

- no MCP implementation
- no duplicate MCP development
- no autonomous execution
- no Cocos automation code
- no real project modification

Human approval remains required before:

- external MCP execution
- Cocos project inspection
- scene, prefab, animation, asset, script, metadata, or build changes
- destructive operations
- release builds

The adapter may propose. Humans approve.
