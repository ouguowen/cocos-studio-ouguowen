# Game Type Templates

Use this file when the team needs a starting production pattern for a specific game type instead of inventing structure from scratch.

## Template law

- Choose the game type by dominant player loop, not by theme.
- Do not force one content model onto every game type.
- Do not borrow monetization or retention patterns before the base loop fits.

## Selection order

1. Name the dominant repeatable player action.
2. Name the main content unit.
3. Name the progression structure.
4. Name whether the game is authored, pool-based, generated, or hybrid.
5. Pick the closest template.

## 1. Level-Based Action Template

- Best for: runner combat, action stages, shooting stages, lane combat, encounter chains.
- Main content unit: stage or chapter level.
- Recommended config center: level, wave, spawn, objective, reward.
- MVP focus: one hero, one enemy family, one victory rule, one fail rule.
- Common danger: overbuilding meta before the action loop feels good.

## 2. Puzzle Or Mechanic-Clear Template

- Best for: match, grid, line, logic, trap, route, switch puzzles.
- Main content unit: puzzle board or puzzle scenario.
- Recommended config center: board layout, mechanic rules, step limit, hint rules, clear conditions.
- MVP focus: one mechanic set, one readable fail state, fast restart.
- Common danger: forcing combat-wave tables onto puzzle logic.

## 3. Roguelite Room-Run Template

- Best for: room combat, card-battle run, dungeon branches, replay-heavy short sessions.
- Main content unit: room, encounter, event node, run seed.
- Recommended config center: room pool, encounter tags, reward choice, modifier tables, progression carry-over.
- MVP focus: one short run, meaningful build variation, restart motivation.
- Common danger: building permanent meta too early and hiding weak run gameplay.

## 4. Tower Defense Or Lane Strategy Template

- Best for: route defense, lane pressure, deploy timing, wave routing.
- Main content unit: map route plus wave package.
- Recommended config center: map nodes, route points, wave schedule, unit cost, tower data, enemy traits.
- MVP focus: one map, one resource model, one readable counterplay set.
- Common danger: too many unit types before route pressure is readable.

## 5. Narrative Exploration Template

- Best for: story rooms, exploration chapters, talk-investigate-progress structures.
- Main content unit: region, room, event chain, chapter objective.
- Recommended config center: location graph, interaction nodes, quest steps, dialogue state, unlock flags.
- MVP focus: one short chapter with clear narrative pull and interaction rhythm.
- Common danger: overbuilding dialogue tools before navigation and objective readability work.

## 6. Idle Or Economy Growth Template

- Best for: idle growth, factory loops, management accumulation, passive progression.
- Main content unit: production chain, upgrade node, session return reward.
- Recommended config center: production formulas, timers, unlock gates, sink tables, reward cadence.
- MVP focus: one production chain, one reinvestment decision, one return hook.
- Common danger: fake depth through too many currencies and tabs.

## 7. Merge Or Collection Template

- Best for: merge board, collection progression, upgrade chain, event board loops.
- Main content unit: board slot, merge recipe, order request, collection target.
- Recommended config center: item tree, spawn source, energy cost, order rewards, storage limits.
- MVP focus: one merge tree, one order loop, one progression hook.
- Common danger: inventory and board rules become unreadable too early.
