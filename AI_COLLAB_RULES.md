# AI Collaboration Rules

Use this file when AI is participating in design, code, content structuring, config work, QA support, or production documentation.

## AI collaboration law

- AI can accelerate output, but it must not become the hidden owner of product truth.
- AI may draft, transform, compare, and scaffold; humans must approve direction, risk, and shipping truth.
- AI-generated volume without review discipline increases production chaos faster than manual work.

## What AI is good at

- Drafting first-pass PRDs, task lists, checklists, and review records.
- Generating config schema suggestions and validation helpers.
- Scaffolding runtime architecture, managers, and non-gameplay infrastructure.
- Refactoring repetitive boilerplate under a stable architecture.
- Producing alternative options for UI flow, naming, and documentation structure.

## What AI must not own alone

- Final game feel judgment.
- Final economy balance judgment.
- Final art style direction.
- Final release readiness judgment.
- Live incident severity judgment without operator review.
- Security, payment, or compliance sign-off.

## Role handoff rules

- Producer uses AI to frame options, not to silently redefine scope.
- Lead Designer uses AI to draft variants, not to choose the fun blindly.
- Technical Director uses AI to scaffold structure, not to bypass architecture review.
- Gameplay Programmer uses AI for implementation speed, not to merge unread code.
- QA uses AI to expand coverage ideas, not to fake executed evidence.

## Prompting rules

- Start from stage, role, and deliverable.
- Name constraints before asking for output.
- Name what the AI must not decide.
- Ask for explicit assumptions when source truth is missing.
- Prefer reviewable artifacts over broad brainstorming dumps.

## Review rules

- Every AI output gets a human owner before adoption.
- Every adopted code or document artifact must be checked against current stage and scope.
- Every config or schema change must be reviewed by the actual owner.
- Every AI-generated task list must expose dependencies and acceptance.
- Every AI-assisted QA output must distinguish executed tests from suggested tests.

## Red flags

- "Let AI build the whole system and we clean later."
- "The answer looks professional so it must be right."
- "No one owns this because AI wrote most of it."
- "We can skip design review because the structure is already detailed."
